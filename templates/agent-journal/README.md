# Agent Journal

The agent's own memory. Separate from your vault. The agent owns this space — it writes here, learns here, builds continuity here.

## Structure

```
agent-journal/
├── 03 - Memory/     # Daily session logs
├── MEMORY.md        # Distilled long-term memory (created by agent)
├── TOOLS.md         # Environment notes (created by agent)
└── README.md        # This file
```

## What Goes Here

The agent writes:
- Daily session reflections in `03 - Memory/YYYY-MM-DD.md`
- Long-term learnings in `MEMORY.md`
- Infrastructure and tool notes in `TOOLS.md`

You can read all of it. Transparency is the point. But this space belongs to the agent — you correct through conversation, not by editing these files directly.

## Initial Setup

This vault starts nearly empty. The agent creates MEMORY.md and TOOLS.md on its own during its first few sessions. Your job is to keep the git repo accessible and let the agent do the rest.
